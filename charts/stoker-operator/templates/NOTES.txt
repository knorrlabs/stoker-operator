== Next Steps ==

1. Create secrets for git authentication and gateway API access:

    kubectl create secret generic git-creds -n <namespace> \
      --from-literal=token=<your-git-token>

    kubectl create secret generic gw-api-key -n <namespace> \
      --from-literal=apiKey=<token-name>:<token-secret>

2. Apply a GatewaySync CR to define the git repository and sync profiles:

    apiVersion: stoker.io/v1alpha1
    kind: GatewaySync
    metadata:
      name: my-sync
    spec:
      git:
        repo: "https://github.com/your-org/ignition-config.git"
        ref: "main"
        auth:
          token:
            secretRef:
              name: git-creds
              key: token
      gateway:
        api:
          secretName: gw-api-key
      sync:
        profiles:
          default:
            mappings:
              - source: "projects/"
                destination: "projects/"
                type: dir
                required: true
            syncPeriod: 30

{{- if .Values.webhook.namespaceSelector.requireLabel }}
3. Label the namespace for sidecar injection:

    kubectl label namespace <namespace> stoker.io/injection=enabled
{{- else }}
3. Sidecar injection is enabled cluster-wide by default.
   Pods with annotation stoker.io/inject: "true" will be injected in any namespace.
   To restrict injection to specific namespaces, set webhook.namespaceSelector.requireLabel=true.
{{- end }}

{{- if .Values.rbac.autoBindAgent.enabled }}
4. Agent RBAC is managed automatically. The controller creates RoleBindings in each
   namespace where GatewaySync CRs exist. To manage RBAC manually, set rbac.autoBindAgent.enabled=false.
{{- else }}
4. Grant the agent RBAC in each namespace where gateways run:

    kubectl create rolebinding stoker-agent -n <namespace> \
      --clusterrole=stoker-agent \
      --serviceaccount=<namespace>:<gateway-service-account>
{{- end }}

5. Add the injection annotations to your Ignition gateway pod template:

    podAnnotations:
      stoker.io/inject: "true"
      stoker.io/cr-name: "my-sync"
      stoker.io/profile: "default"
      # stoker.io/gateway-name: "my-gw"  # optional — defaults to app.kubernetes.io/name label or pod name

{{- if and .Values.webhook.enabled .Values.certManager.enabled }}

Ensure cert-manager is installed: https://cert-manager.io/docs/installation/
{{- end }}
